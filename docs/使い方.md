# SampleEqSim 使い方ガイド

**バージョン:** 1.0.0  
**対象:** SampleEqSim.Equipment / SampleEqSim.Host

---

## 1. 前提条件

| 項目 | 要件 |
|------|------|
| OS | Windows 10 / 11 (64bit) |
| 開発環境 | Visual Studio 2022 (17.8 以降) |
| .NET | .NET 8.0 SDK |
| Visual Studio ワークロード | .NET デスクトップ開発 |

---

## 2. セットアップ

### 2.1 ソリューションを開く

1. `SampleEqSim.sln` を Visual Studio で開く
2. NuGet パッケージの復元を行う  
   (`ソリューション` 右クリック → `NuGet パッケージの復元`)
3. ソリューション全体をビルド (`Ctrl+Shift+B`)

### 2.2 スタートアッププロジェクトの設定

複数プロジェクト同時起動の設定:

1. ソリューション右クリック → **スタートアッププロジェクトの設定**
2. **マルチ スタートアップ プロジェクト** を選択
3. `SampleEqSim.Equipment` → **開始**
4. `SampleEqSim.Host` → **開始**
5. **OK** をクリック

> 順番は Equipment を先に設定することを推奨 (サーバー側が先に起動するため)

## 4. 基本的な通信テスト手順

### Step 1: アプリ起動

1. `F5` でデバッグ開始 (Equipment・Host が同時起動)
2. Equipment の状態バーが `COMM: DISABLED` であることを確認
3. Host の接続状態が `CONNECTING` → `SELECTED` に変わることを確認
4. Equipment の状態バーが `COMM: WAIT_CR_FROM_HOST` に変わることを確認

### Step 2: 通信確立

Host 側で操作:

1. **[S1F1 Are You There]** をクリック
   - Equipment のモデル名・SoftRev が表示される
2. **[S1F13 通信確立]** をクリック
   - Equipment の `COMM` が `COMMUNICATING` に変わる
3. **[S1F17 ON-LINE]** をクリック
   - Equipment の `CTRL` が `ONLINE_REMOTE` に変わる

### Step 3: データ収集設定

Host 側で操作:

1. **[S2F33 Define Report]** をクリック
   - RPTID=1 に SVID 101(温度)・102(圧力)・103(ロットID) を登録
2. **[S2F35 Link Event Report]** をクリック
   - CEID 101(ProcessStarted)・102(ProcessCompleted) に RPTID=1 をリンク
3. **[S2F37 Enable All Events]** をクリック
   - 全コレクションイベントを有効化

### Step 4: 処理シミュレーション

Equipment 側で操作:

1. `ロットID` に任意の文字列を入力 (例: `LOT-001`)
2. **[値を更新]** をクリック
3. **[LotStarted]** をクリック → Host のイベントログに `CEID=103` が表示
4. **[EXECUTE]** ボタンをクリック → 処理状態が `EXECUTING` に変化
   - Host のイベントログに `CEID=101` (ProcessStarted) が表示される
5. 温度・圧力スライダーで値を変更し **[値を更新]**
6. Host 側で **[S1F3 SV取得]** をクリック → 現在の SV 値を確認
7. **[IDLE]** をクリック → 処理状態が `IDLE` に変化
   - Host のイベントログに `CEID=102` (ProcessCompleted) が表示

### Step 5: アラームテスト

Equipment 側で操作:

1. **[AL1 SET]** をクリック
   - Host のイベントログに `[ALARM SET] ALID=1 低圧縮空気検出` が表示
2. **[全クリア]** をクリック
   - Host のイベントログに `[ALARM CLR]` が表示

## 5. ホストコマンド (S2F41) によるリモート制御

Host 側の「装置制御」パネルから送信できます。

| コマンド | 操作 | 動作 |
|---------|------|------|
| START | テキストボックスに `START` → [送信] | 処理状態を EXECUTING に遷移 |
| STOP | テキストボックスに `STOP` → [送信] | 処理状態を IDLE に遷移 |
| PAUSE | テキストボックスに `PAUSE` → [送信] | 処理状態を PAUSE に遷移 |
| RESUME | テキストボックスに `RESUME` → [送信] | 処理状態を EXECUTING に再開 |
| LOCAL | テキストボックスに `LOCAL` → [送信] | 制御状態を OnlineLocal に遷移 |
| REMOTE | テキストボックスに `REMOTE` → [送信] | 制御状態を OnlineRemote に遷移 |

## 6. 設定変更

### 6.1 接続先 IP・ポートの変更

Equipment を別 PC で動作させる場合:

**Host 側 `appsettings.json`:**
```json
{
  "secs4net": {
    "IsActive": true,
    "IpAddress": "192.168.1.100",  ← Equipment の IP に変更
    "Port": 5000
  }
}
```

**Equipment 側 `appsettings.json`:**
```json
{
  "secs4net": {
    "IsActive": false,
    "IpAddress": "0.0.0.0",
    "Port": 5000
  }
}
```

### 6.2 装置モデル名・バージョンの変更

**Equipment 側 `appsettings.json`:**
```json
{
  "Equipment": {
    "ModelName": "MyEquipment",
    "SoftRev": "2.0.0"
  }
}
```

### 6.3 タイムアウト値の変更

T7 (通信確立タイムアウト) を変更する場合:

**Equipment 側 `appsettings.json`:**
```json
{
  "secs4net": {
    "T7Timeout": 30000
  }
}
```

または Host 側から `S2F15` (EC 設定) で ECID=1 (EstablishCommunicationsTimeout) を変更。

## 7. ログの見方

### 7.1 通信ログ

| プレフィックス | 意味 |
|--------------|------|
| `SND >> S1F13` | ホスト→装置 送信 |
| `RCV << S1F14` | 装置→ホスト 受信 |
| `[HSMS]` | HSMS レイヤーの状態変化 |
| `[ComState]` | 通信状態遷移 |
| `[CtrlState]` | 制御状態遷移 |
| `[ProcState]` | 処理状態遷移 |
| `[CMD]` | ホストコマンド受信 |
| `[ALARM SET/CLR]` | アラーム発生/クリア |
| `[TERMINAL]` | 端末メッセージ |
| `[ERR]` | エラー |
| `[WARN]` | 未実装メッセージ受信 |

### 7.2 ログ色 (Host 側)

| 色 | 意味 |
|----|------|
| 青 | 送信メッセージ |
| 緑 | 受信メッセージ |
| 赤 | アラーム |
| オレンジ | エラー |
| 白 | システム情報 |

## 9. よくある問題

### Q. Host が接続できない

- Equipment が先に起動していることを確認
- ファイアウォールで Port 5000 (TCP) が許可されているか確認
- `appsettings.json` の IP・ポートが一致しているか確認

### Q. 通信確立後に S6F11 (イベント) が届かない

以下の順序で設定が必要:
1. `S2F33` でレポートを定義
2. `S2F35` でイベントとレポートをリンク
3. `S2F37` でイベントを有効化

### Q. appsettings.json が見つからないエラー

Visual Studio でプロジェクトの `appsettings.json` のプロパティを確認:
- **出力ディレクトリにコピー:** 「常にコピーする」に設定

### Q. 実機装置と接続したい

**Host Simulator を実機装置に接続する場合:**
- Host の `appsettings.json` の `IpAddress` を実機の IP に変更
- `IsActive: true` のまま使用

**実機ホストに Equipment Simulator を接続する場合:**
- Equipment の `appsettings.json` の `Port` を実機ホストが待受けているポートに変更
- 実機ホストが Active の場合、Equipment は Passive のままで動作

## 10. 拡張方法

### 新しいステータス変数を追加

`GemEquipmentModel.cs` の `InitializeGemData()` に追加:

```csharp
StatusVariables[200] = new StatusVariable(200, "MyVariable", "A", () => "値", "unit");
```

### 新しいアラームを追加

```csharp
Alarms[10] = new AlarmDefinition(10, "MY_ALARM", "アラームテキスト", AlarmCategory.Warning);
```

### 新しいコレクションイベントを追加

```csharp
CollectionEvents[200] = new CollectionEventDefinition(200, "MyEvent");
EnabledEvents[200] = true;
```

イベント送信:
```csharp
await _model.SendCollectionEventAsync(200);
```

### 新しいリモートコマンドを追加

`GemEquipmentModel.cs` の `HandleS2F41()` の switch 式に追加:

```csharp
"MYCOMMAND" => ExecuteMyCommand(),
```
